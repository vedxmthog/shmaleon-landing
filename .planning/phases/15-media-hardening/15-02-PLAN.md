<task type="auto">
  <name>Apply width height decoding lazy alt</name>
  <files>index.html, bespoke.html, the-story.html</files>
  <read_first>index.html, bespoke.html, the-story.html, .planning/phases/15-media-hardening/15-PATTERNS.md</read_first>
  <action>
For **each** `<img>` whose `src` starts with **`media/shm-`**:

1. Add **`width="{W}"` `height="{H}"`** where W/H are the natural dimensions from the previous task for that file.
2. Add **`decoding="async"`** to every such `<img>`.
3. **`loading` attribute:**
   - **`loading="lazy"`** for: all **the-story.html** ritual grid images (`shm-story-sourcing`, `shm-story-infusion`, `shm-story-presentation`), **the-story** handover image (`shm-story-handover`), **index.html** object sections (`shm-ingot`, `shm-pulse`, `shm-heart-cake`, `shm-delivery`), and **bespoke.html** product tiles (`shm-ingot`, `shm-pulse`, `shm-heart-product`).
   - **Omit `loading="lazy"`** (or set `loading="eager"` only if you must override browser default) for: **index.html** hero (`shm-index-hero`), **bespoke.html** hero (`shm-bespoke-hero`), **the-story.html** hero (`shm-story-hero`) — these are above-the-fold LCP candidates.

4. **`alt` text (A11Y-01):**
   - If `alt=""` and `data-alt` is present: set **`alt`** to a **short** English phrase (max ~120 chars) summarizing `data-alt` (e.g. hero: `Ornate dark gift box in spotlight` — adjust to match brand tone).
   - For **bespoke** tiles that already have non-empty `alt` (`The Ingot`, etc.): **keep** those alts unless they are clearly wrong; ensure they are non-redundant with adjacent headings if screen reader experience duplicates.
   - Do **not** duplicate paragraph-length `data-alt` verbatim in `alt`.

5. **Leave `data-alt` attributes as-is** unless they violate HTML validity (they are valid custom data attributes).

Do not change **`src`** paths in this task except to fix extension typos if needed.
  </action>
